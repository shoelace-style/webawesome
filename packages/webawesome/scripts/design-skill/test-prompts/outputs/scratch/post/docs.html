Typography

Most websites are typeset like tax forms. Halftone is for the rest of us — for designers who think a paragraph longer than two alphabets is a confession, that real ligatures matter, and that a baseline grid is the difference between a layout and a list of things on a page. The defaults here have opinions. The rest of this page is where they get explained.

Type scale #

Halftone uses a modular scale of 1.125, which is the minor second in musical terms. That's a small ratio — deliberately so. A scale of 1.333 puts your h2 in a parade. 1.5 puts it on a billboard. 1.125 puts it on a page, where it belongs.

You read the scale up from the body size. The body stays 1rem. Each step is the previous step times 1.125. By the time you reach the h1 you're at 1.602rem — bigger than the body, smaller than a shout. Six steps cover ninety percent of editorial work; if you need more, you don't need more steps, you need a magazine layout, and that's a different framework.

Overriding the scale. Set --halftone-scale on :root. 1.2 (major second) gives a touch more contrast. 1.25 (minor third) is louder still. We've tested up to 1.333 and stopped — past that, your h1 starts to sound less like a heading and more like a megaphone.

Vertical rhythm #

A baseline grid is the set of imaginary horizontal lines body text aligns to — the same way a sheet of ruled paper works, only invisible. Halftone sets that grid to 1.5rem (24px on a 16px base) and every block-level element snaps to a multiple of it. Paragraph margins, heading margins, list spacing, blockquote insets: all 1.5rem, 3rem, or 4.5rem. Nothing in between.

Why bother? Because the result is a page where text blocks line up across columns, even when those columns hold different things — a heading on the left, body on the right, a sidebar in the gutter. Without the grid, the columns drift relative to each other and the page reads as a collage. With it, the page reads as a page. 

:root {
  --halftone-baseline: 1.5rem;
  --halftone-leading:  1.55;
}

p, ul, ol, blockquote {
  margin-block: var(--halftone-baseline);
  line-height:  var(--halftone-leading);
}

/* Snap headings to a tighter leading so they don't break the rhythm. */
h1, h2, h3, h4, h5, h6 {
  line-height: var(--halftone-heading-leading, 1.15);
  margin-block: calc(var(--halftone-baseline) * 2) var(--halftone-baseline);
}
The most common pitfall: forgetting to set line-height on headings. If h1–h6 inherit the body line-height of 1.5, the leading swallows large type and the descenders knock into the next line. Pin heading leading somewhere between 1.1 and 1.25. The prebuilt stylesheet does this for you; if you're picking and choosing modules, it's on you.

Print-safe fonts #

PDF embedding is where ambitious web type goes to die. The variable font with twelve axes and twenty stylistic sets that looks like a Brooklyn studio's rebrand in Chrome will, in any PDF library written before 2024, get flattened to a single weight, embedded as a bitmap, or substituted for Helvetica entirely. The web is one rendering target. Print is another. Halftone's default stack is built for both, which means static fonts in the print stylesheet: no variable axes, no color glyphs, no web-only providers that load fonts via JavaScript after the DOM is ready — by the time the PDF generator sees your page, those fonts have evaporated.

Variable fonts in print stylesheets are a trap. Chrome flattens them to a single instance, which is usually fine. Safari flattens them to outline data and triples your file size. Firefox sometimes drops the font entirely and substitutes Times. There is no portable answer — use static fonts inside your @media print block and don't gamble on the renderer.

Safe choices

  • Iowan Old Style (system)
  • Charter (system)
  • Source Serif 4 (static OTF)
  • IBM Plex Sans (static OTF)
  • JetBrains Mono (static OTF)

Avoid for print

  • Variable fonts of any kind
  • Color & SVG glyph fonts
  • Google Fonts loaded at runtime
  • Adobe Fonts via JS embed
  • Anything without a desktop license

None of this is forever. Variable-font support in print renderers is improving, and in two years the danger callout above may read as superstition. Until then, Halftone ships static, embeds clean, and prints what you see.
