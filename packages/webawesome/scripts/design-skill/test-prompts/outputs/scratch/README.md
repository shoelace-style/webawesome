# Scratch outputs

A free-for-all directory for **one-off views generated by the `webawesome-design` skill** — exploring what the skill produces for prompts you don't (yet) want as part of the regression corpus.

## Suggested workflow

1. **Open a fresh Claude Code window.** Skills are global at `~/.claude/skills/`, so `webawesome-design` and `webawesome` are present in any session — and a fresh session is the cleanest test (no editorial bias from a session that's been editing the skill itself).
2. **Give it a single brief plus a save path.** Example:

   > Using Web Awesome, build a settings panel for notification preferences — five toggle rows with labels, descriptions, and `<wa-switch>` controls. Save the complete HTML to `/Users/brian/Projects/shoelace-style/webawesome-app/webawesome/packages/webawesome/scripts/design-skill/test-prompts/outputs/scratch/settings.html`.

3. **Open the file.** `open <path>` on macOS, or via the static-server pattern in the parent `test-prompts/README.md`.
4. **Iterate or move on.** If the output is a one-off, leave it. If it surfaces a failure mode worth catching every time, **promote the prompt** into the regression corpus at `test-prompts/NN-name.md` and add it to the rotation.

## What goes here vs. the regression corpus

- **Here (scratch/):** quick exploration, design ideas, "what would the skill do with X?" outputs. No commitment to keep them.
- **In the regression corpus (`test-prompts/*.md` + `outputs/{baseline,post-edit-N}/`):** prompts that exercise specific failure modes we want the skill to keep catching across edits. Stable, paired with rubric notes, committed.

## Git

`*.html` here is intentionally throwaway. If you want this directory gitignored long-term, add `outputs/scratch/*.html` to `.gitignore` and keep this README tracked. Until then, commit only the outputs you want to keep as exemplars.
